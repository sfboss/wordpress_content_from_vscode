## Choose the smallest useful implementation

The offering is designed to begin with a fit check or a non-production pilot. Production access is a separate decision.

### Readiness Review

A short review of current change practices, GitHub ownership, security constraints, and team readiness.

### Repo Starter Kit

Salesforce project scaffold, GitHub Action templates, secrets checklist, ignore patterns, and a dev-org validation runbook for a team implementing internally.

### Dev Org → GitHub Pilot

A working private repository, development-org authorization, scheduled metadata snapshot, initial diff history, and a recorded handoff.

### Production + Team Handoff

Production authorization strategy, workflow hardening, notifications, credential rotation procedure, and team enablement after the pilot is proven.

## Optional modules

- Slack notifications for workflow results
- Google Drive destination for approved exports and reports
- Colab operator runbook
- Certificate-based OAuth hardening
- GitHub Codespace starter environment
- Deployment validation and release evidence

Data export, compliance-specific controls, managed restore expectations, and complex deployment orchestration require separate scope.
